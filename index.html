For this project my aim was to create a mobile game app. The main task of the game developer and designer is to entertain the player, regardless of the device, by building a well-thought-out structure that includes the most important elements. I used Adobe XD for this project.

My hypothetical game is an exploration game, where the player could explore different locations and items. The game is visual and includes minimal text. The interactions are touch screen based, the user has to press on buttons in the menu section and on different in-game objects. The product does not need internet connection to work properly. The app works on mobiles, both on Android and iOS, same goes for working with tablets.

To design the game, first, I made a quick sketch of the user flow map. I decided to make my app vertical, with no need for the user to move their phone into horizontal state to play it. The game would have an introduction page, where one button would lead the user to the menu. The menu would have 4 different options for the user to press: Start, Load Save, Settings, Quit. It is a standard menu that is instinctual nowadays to anyone with a smartphone. The player will see familiar icons, layouts and section names.

I kept it simple. Less is more. The screen of a phone is small and does not require unnecessary visual noise. I took note of the important elements and excluded those that are not needed in terms of functionality or stylistic design. I simplified as much as possible, adapting the design and getting rid of unnecessary stylistic elements. Reduced cognitive load.

The next thing I did is I made an illustration of the environment of my potential game - an abandoned, overgrown castle hidden in the trees with a trail leading towards it and on the way to the castle one might notice a treasure chest. My plan is to make the castle and the chest clickable so the user can see what is inside.

The Item Stats would show different items stored inside objects. The main objective of the game is to explore the world and open new locations while finding new objects. If the user wants to see what is inside the castle, they must first open the treasure chest to find a key to the castle and so on.

The GUI (graphical user interface) will be simple, a “health bar” of the user and a button (diogetic) for the user to see the in-game menu that consists of Settings and Quit Game options (non-diogetic). The Setting contain the options of changing the loudness of music and sound with sliders. As screen space is limited and game content gets bigger, sliders are a handy feature to save space. The design of the app is easy and accesable, the design is similar and consistent to all the other mobile apps (recognition over recall).

My app offers a unique experience for players to explore environments and pick up different items during their adventure!

References: Hodent, Celia. The Psychology of Video Games. 1st ed., Routledge, 2020.
